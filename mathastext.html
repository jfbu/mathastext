Optimal typographical results for documents containing mathematical symbols can only be hoped for with math fonts specifically designed to match a given text typeface. Although the list of freely available math fonts for (PDF)LaTeX is slowly expanding (Computer Modern, AMS fonts, PX fonts, TX fonts, Fourier-GUTenberg, Math Design, Kepler Project, newtx, mathabx, others ... ; and ([2012/10/27]) on the Unicode side (XeTeX/LuaTeX): Asana-Math, XITS, Latin Modern Math, TeX Gyre Pagella Math, TeX Gyre Termes Math) it remains limited. Chances are you will not find a math font which fits well with your favorite text font. Try out mathastext: it will simply use the text font also for the math! (additionally to Latin letters, digits, punctuation signs, a few math symbols available in the ascii set, such as +,–,<,>,=, will also be picked up from the text font.) Often this gives quite satisfying results especially if, like the author, you aim at a very unified look for the document, and in particular prefer upright to slanted shapes for math (mathastext does have an option to maintain the usual slanted shape).

The package originates in some (even worse) macros I had been using many years ago to produce handouts as if done on a typewriter, or better said, to produce documents with a somewhat rough, but unified look, done by a real human and meaning to convey some message to real humans. The hope was to coerce the reader into making a real effort at assimilating the entirety of the material, text and math, not knowing in advance which piece would prove to be more important than the others. In a well-crafted mathematical document, every single part is as important as any other one, or, rather, it is up to the reader to uncover the web of links between atoms.

The example on the side uses a typewriter font which well illustrates the purpose. But mathastext will work with all kinds of fonts and can produce less militant math: see the examples (be patient, this page with its embedded png images weighs circa 5 Mo!) A notable feature of mathastext is to give a simple interface to using many distinct math fonts in the same document: special example.

  • Installation:
    1. using mathastext.tds.zip: run unzip mathastext.tds.zip -d <foo> where foo is the TDS compliant destination folder.
    2. manually: tex mathastext.dtx to extract the package file mathastext.sty, some test files, and also mathastext.tex. There one can customize the global font size or paper dimensions for producing the documentation with or without source code included via latex mathastext.tex (thrice) and dvipdfmx mathastext.dvi. It is also possible to run directly pdflatex mathastext.dtx. Then move the various files to the correct locations...

    • (1.3c) mechanism of "inheritance" in the declaration of mathastext math versions; starred variant of \MTversion leaves untouched the text font.
    • (1.3b)
      1. added a starred variant to \MTversion which tells mathastext to only do the math set-up and not modify the text fonts.
      2. added second optional version name argument to \Mathastext and to \MTDeclareVersion, to transfer settings for things not otherwise changed by mathastext from a math version to the one declared. This is mainly for helping set in bold face also symbols and large symbols when the used font for letters has been chosen bold in the declared version.
    • (1.3a)
      1. new command \MTsetmathskips allows to specify on a letter by letter basis some extra skips to be automatically inserted in math mode before and after each occurrence of the given letter,
      2. the package when running under lualatex requires it in a version at least as recent as the one distributed with TL2013,
      3. the documentation is better hyperlinked.
    • (1.2f) "change log" added to the user manual. Minor code improvements.
    • (1.2e) remaining technical issues introduced with the version 1.2 have been solved; the documentation and the test files have been extended.
    • (1.2d) an incompatibility with amsmath introduced in 1.2 is fixed.
    • (1.2b) The package achieves a slight improvement in the positioning of subscripts when using an upright text font for letters in text mode and it corrects bugs with active characters which were introduced in 1.2.
    • (1.2)
      1. A new command sets up the amount of space to be automatically inserted before the derivative glyph (useful when using an upright font).
      2. The scope of the math alphabets has been extended to apply to the non-alphabetical characters, and also to operator names. In usual math mode $\mathbf{(\sin(n!)<\cos(x)?)}$ changes only the font for n and x. With mathastext and if not told otherwise, /everything/ in the formula above will respond to \mathbf (and the automatic spaces are left untouched).
      3. The documentation has been improved. The package file is self-extracting from the dtx, and four additional test files are also produced during `latex mathastext.dtx'.
    • (1.15g) improvements to the documentation; modifications of the \#,\&,\$ and \% related code which was working but admittedly a bit strange; changes to the behavior of the math alphabets related options; general clean up of the code and of its comments.
    • (1.15e) commands to add some spacing after ∃ and ∀, additions to the subdued option implementation.
    • (1.15d) additional fixes to problems related to the amsmath package, especially under the Unicode engines XeTeX or LuaLaTeX.
    • (1.15c) each math version can have its own LGR font for the Greek letters in math mode,
    • (1.15) introduction of the subdued package option, which allows to let mathastext act only on some limited portions of the document; and a fix to a problem of compatibility with amsmath when using the Unicode engines XeTeX or LuaLaTeX.

  • Random bits of info:
    • The sans-serif member of the document font family gives often good results when used in math-mode, in parallel with the text in serif. 
      \renewcommand{\familydefault}{\sfdefault}
\usepackage{mathastext}
\renewcommand{\familydefault}{\rmdefault}
      It is then even less recommended to use the italic option (matter of taste!).
    • How get in math mode the upright Greek letters from Computer Modern? Include in your preamble: 
      \usepackage{my_nice_font}
\renewcommand{\familydefault}{cmr} % or cmss or cmtt for sans resp. mono
\usepackage[LGRgreek, further options]{mathastext}
\renewcommand{\familydefault}{\rmdefault} 
\Mathastext
\begin{document}
      The Greek letters will be from the LGR encoded CM font (and will be in upright shape, except if italic was among the mathastext options).
    • The previous trick works for (pdf)LaTeX as well as for XeLaTeX or LuaLaTeX. For XeLaTeX the mathspec package is a (far) better choice, but if you wish you can still use mathastext for its other aspects (math versions...).
    • The LGRgreek option (\usepackage[LGRgreek]{mathastext}) picks up the Greek letters in LGR encoding from the default font family at the time of loading mathastext. Starting with version 1.15c it is possible to have distinct Greek fonts in each math version. I mentioned already above that the CM/LM fonts are available (in Type 1 version for inclusion in pdf files) in LGR encoding for serif, sans-serif, and typewriter families (they actually belong to the "cbfonts".) Other available fonts in standard TeX distributions: the Greek Font Society fonts (such as GFS Didot), the epigrafica and kerkis packages, the txfontsb package which extends the txfonts package with LGR-encoded Greek letters, the droid, comfortaa, dejavu, and opensans package.
    • Note that mathastext makes proviso only for monotonic Greek letters (in math mode; many of these packages are designed for polytonic Greek in text mode). After loading mathastext with option LGRgreek one should take note that \varphi and other `var' letters (except \varsigma) are not provided by this encoding, so the package does not make any specific assignments to such `var' macros.
