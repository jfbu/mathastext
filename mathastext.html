\usepackage[T1]{fontenc}
\usepackage[german]{babel}
\usepackage[variablett]{lmodern}
\renewcommand\familydefault\ttdefault
\usepackage[LGRgreek]{mathastext}
\MTgreekfont{lmtt}\Mathastext

Optimal typographical results for documents containing mathematical symbols can only be hoped for with math fonts specifically designed to match a given text typeface. Although the list of freely available math fonts (alongside the Computer Modern and AMS extension fonts) is slowly expanding (fourier, kpfonts, mathdesign, pxfonts, txfonts, and others...) it remains limited, and the situation is even worse with Unicode fonts (XeTeX/LuaTeX). So if you can't find a math font which fits well with your favorite text font, try out mathastext: it will simply use the text font also for the math! (only the few math symbols available in the ascii set, such as +,–,<,>,=, will be picked up from the text font.) Often this gives quite satisfying results especially if, like the author, you aim at a very unified look for the document, and in particular prefer upright to slanted shapes for math (mathastext does have an option to maintain the usual slanted shape).

The package originates in some (even worse) macros I had been using many years ago to produce handouts as if done on a typewriter, or better said, to produce documents with a somewhat rough, but unified look, done by a real human and meaning to convey some message to real humans. The hope was to coerce the reader into making a real effort at assimilating the entirety of the material, text and math, not knowing in advance which piece would prove to be more important than the others. In a well-crafted mathematical document, every single part is as important as any other one, or, rather, it is up to the reader to uncover the web of links between atoms.

The example on the side uses a typewriter font which well illustrates the purpose. But mathastext will work with all kinds of fonts and can produce less militant math: see the examples (be patient, this page with its embedded png images weighs circa 4,8 Mo!) A notable feature of mathastext is to give a simple interface to using many distinct math fonts in the same document: special example.

  • Additions to the documentation:
    • How to get the Greek letters from Computer Modern in upright shape? Example: 
      \usepackage[T1]{fontenc}
\usepackage[basic,LGRgreek]{mathastext}
\usepackage[lf]{electrum}
\Mathastext 
\begin{document}
      This will set up to use ADF-Electrum Latin letters with lining figures also in math, together with the upright Greek letters from Computer Modern.
    • The sans-serif member of the document font family gives often good results when used in math-mode, in parallel with the text in serif. Use this: 
      \renewcommand\familydefault\sfdefault
\usepackage{mathastext}
\renewcommand\familydefault\rmdefault
      It is then even less recommended to use the italic option (matter of taste!).
    • The LGRgreek option (\usepackage[LGRgreek]{mathastext}) picks up the Greek letters in LGR encoding from the default font family at the time of loading mathastext. With the latest version of the package it is now possible to pick up distinct Greek fonts, depending on the math version. It is useful to know that the CM/LM fonts (and CM-LGC, package cmlgc) are available in LGR encoding for serif, sans-serif, and typewriter families. Other available fonts in standard TeX distributions:the Greek Font Society fonts (such as GFS Didot), the epigrafica and kerkis packages, the txfontsb package which extends the txfonts package with LGR-encoded Greek letters, the droid, comfortaa, dejavu, and opensans package.
    • Note that mathastext makes proviso only for monotonic Greek letters (in math mode; many of these packages are designed for polytonic Greek in text mode). After loading mathastext with option LGRgreek one should take note that \varphi and other `var' letters (except \varsigma) are not provided by this encoding, so the package does not make any specific assignments to such `var' macros.

  • Known limitations and bugs:
    • (2012/10/12) [update 2012/10/13: v1.15d fixes this] There are some left-over issues (from v1.14 of April 2011) in the code of mathastext when it is used with the unicode engines XeTeX and Lualatex, and especially regarding some interactions with the amsmath package, which I am currently reviewing. For XeTeX, everything is fixed, but I still have some progress to realize with Lualatex. If you encounter a problem with the current version of mathastext, try the option nominus, or perhaps only noendash.
    • Some characters from fonts having complex ligature rules may not display correctly in math mode (here is an example).
    • Although the subdued option claims to let mathastext act only on some limited parts of the document, it just hides to the best possible the sinister dealings done by mathastext: it does not really cancels them. To do that one needs to also use basic, defaultmathsizes, and defaultalphabets (and nohbar, and defaultimath, and ... no that's all) but then the influence of mathastext when activated in a math version is lessened.
